3. External

file code

html link rel="stylesheet" href="css/style.css"
  1. css > 스타일 규칙 선언
  2. html > head 내부에 "link" tag 선언
  3. html > "href": scc 경로
  4. html > "rel": 파일과 문서의 관계 명시. CSS == "stylesheet"
외부 스타일 시트 파일 이용
  • 스타일 규칙을 별도의 외부 파일을 만들어 넣늠

  • 파일 관리 편하고 용량이 작음
  • 주로 사용되는 방법

선택자 중 가장 기본이 되는 선택자
== 태그 선택자

h1{color:yellow;}
h2{color:yellow;}
h3{color:yellow;}
...


  • 선택자 부분에 태그 이름 명시
  • 문서 내 선택자에 해당하는 모든 요소에 스타일 규칙 적용

그룹화

쉼표를 이용해 그룹화 가능

h1,h2,h3,h4,h5,h6{color:yellow;}


전체 선택자

*{color:yellow;}

  • * 별표, asterisk 기호: 문서 내 모든 요소 선택
  • 한 번의 선언만으로 문서 내 모든 요소에 스타일 규칙 적용
  • 편리, 성능에 좋지 않으므로 지양

선언 그룹화

h1{color:yellow; font-size: 2em; background-color: gray;}


선택자, 선언 동시 그룹화

h1, h2, h3, h4, h5, h6{color:yellow; font-size: 2em; background-color: gray;}

'dt'에 컬러 지정 > "CSS"에도 적용

4: 선택자 2

class 선택자

요소에 구애받지 않고 스타일 규칙을 적용할 수 있는 가장 일반적인 방법

HTML 수정 > class 속성 추가
글로벌 속성이므로 어느 태그에서도 사용 가능
class > 값: class 선택자로 해당 요소에 스타일 규칙 적용


css > .foo{font-size:30px;}

html > |p class="foo"| ... |/p

  • 클래스 선택자 사용: 맨 앞에 . 마침표 찍어야 함
  • 어느 요소든지 class 속성값이 'foo'인 요소는 해당 스타일 규칙 적용

다중 class

여러 개의 class 값: 공백으로 구분

p class="foo bar"| ... |\p
.foo{font-size: 30px}
.bar{color:blue;}

id 선택자

class 선택자와 유사

. 대신 # 해시 기호 사용
class 속성 대신 id 속성

css > #bar{background-color: yellow;}
html > p id="bar"| ... |\p

class 선택자와의 차이점

  1. . 기호가 아닌 # 기호 사용
  2. 태그의 class 속성이 아닌 id 속성을 참조
  3. 문서 내에 유일한 요소에 사용
  4. 구체성

가장 큰 차이점: id는 문서 내에서 유일해야 함

적용 가능한 요소는 단 하나
구체성의 값이 다름

들어가기 전에

종류에 상관 없이 여러 선택자 조합 가능

5: 선택자 3

선택자의 조합

- 요소와 class의 조합
p.bar{...}

- 다중 class -
.foo.bar{...}

- id와 class의 조합
#foo.bar{...}

  1. 요소와 클래스를 조합한 경우
    | p |이면서 class 속성에 bar가 있어야 적용
  2. 다중 클래스의 경우
    class 속성에 foo, bar 모두 있어야 적용
  3. id와 class 조합
    id==foo, class==bar인 요소에 적용

속성 선택자

단순 속성으로 선택

css > p[class]{color:silver;}
        > p[class][id]{text-decoration: underline;}

html > p class="foo"| Hello |/p
        > p class="bar"| CSS |/p
        > p class="baz" id="title"| HTML |/p

선언: 대괄호, 대괄호 내 속성 이름

  1. p, class: color:siver
  2. p, class, id: text-decoration:underline

속성 값은 상관하지 않음

정확한 속성값으로 선택

css > p[class="foo"]{color:silver;}
        > p[id="title"]{text-decoration:underline;}

  • p[class="foo"]: p, class==foo면 적용
  • p[id="title"]: p, id==title이면 적용

부분 속성값으로 선택

속성명과 속성값 사이에 사용되는 기호에 따라 동작이 다름

  • [class~="bar"]: class 속성 값이 공백으로 구분한 "bar" 포함 요소 선택
  • [class^="bar"]: class 속성 값이 "bar"로 시작하는 요소 선택
  • [class$="bar"]: class 속성 값이 "bar"로 끝나는 요소 선택
  • [class*="bar"]: class 속성 값이 "bar"문자를 포함하는 요소 선택

들어가기 전에

조합자, 결정자: 문맥이나 요소의 구조를 기반으로 선택자를 조합하는 것
문서 구조를 이용해 좀 더 유연하게 요소 선택 및 스타일 적용 가능

6: 문서 구조 관련 선택자

문서 구조의 이해

  • html
  • body
  • div
  • h1 | span | /span | /h1
  • /div
  • p | /p
  • /body
  • /html

부모와 자식

부모요소: 그 요소를 포함하는 가장 가까운 상위 요소
        그 요소의 부모 요소는 단 하나

자식요소: 자식 요소는 여러개일 수 있음

태그 부모요소 태그 자식요소
body html html body
div body body div, p
h1 div div h1
span h1 h1 span
p body body div, p

조상과 자손

조상요소: 그 요소를 포함하는 모든 요소
        부모 요소를 포함하여 여러개일 수도 있음

자손요소: 요소가 포함하는 모든 요소

태그 조상요소 태그 자손요소
body html html body, div, h1, span, p
div html, body body div, h1, span, p
h1 html, body, div div h1, span
span html, body, div, h1 h1 span
p html, body body div, h1, span, p

형제

같은 부모를 가진 요소: div, p

인접한 형제 관계: 요소 중 바로 뒤에 이어 나오는 요소
        p가 div에 인접한 형제 요소

문서 구조 관련 선택자

자손 선택자

div span{color:red;}

선택자 사이 기호 없이 공백
div의 자손 요소인 span 선택

자식 선택자

div > h1{color:red;}

선택자 사이 > 꺽쇠 기호 사용
꺽쇠 기호와 선택자 사이에 공백은 상관 없음
div 자식요소인 h1 선택

인접 형제 선택자

div + p{color:red;}

선택자 사이 + 기호
자식선택자와 마찬가지로 공백은 상관 없음
형제관계이면서 바로 뒤에 인접한 요소 선택

- body 요소와 자식인 div 요소의 자손인 table 요소 바로 뒤에 인접한 ul 요소 선택
body > div table + ul{ ... }

문서구조 관련 선택자는 더 복잡하게 사용 가능

요소가 많이 나열되어 있더라도 제일 우측 요소가 실제 선택 요소

7: 가상 선택자 1

가상 클래스 pseudo class

미리 정의해놓은 상황에 적용되도록 약속된 보이지 않는 클래스

약속된 상황이 되면 브라우저 스스로 클래스 적용

"p" 태그가 있다고 가정: p에 마우스 커서를 올렸을 때만 특정 스타일 적용
만약 가상 클래스가 없다면 ...

  1. 임의 클래스 선택자를 선언하여 특정 스타일 규칙 만듦
  2. p 요소에 커서가 올라가면 p 요소에 클래스 집어넣기
  3. p 요소에 커서가 빠지면 p 요소에 클래스 삭제
2, 3번은 동적으로 변화되어야 하는데, HTML과 CSS는 정적인 언어이므로 처리 불가

CSS에서 흔히 사용되는 여러 패턴을 미리 정의, 가상 클래스로 제어
:pseudo-class{
property:value;
}

가상 클래스는 : 콜론 기호 사용

예시: ":hover" 가상 클래스 선택자를 이용해 스타일 규칙 설정
hover: 마우스 커서가 올라갔을 때 적용이 되도록 정의

문서 구조와 관련된 가상 클래스

  • :first-child: 첫 번째 자식 요소 선택
  • :lastt-child: 마지막 자식 요소 선택

HTML >
ul
li | html | /li
li | css | /li
li | js | /li
/ul

CSS >
li: first-child{color:red;}
li: last-child{color:blue;}

앵커 요소와 관련된 가상 클래스

  • :link: 하이퍼링크이면서 아직 방문하지 않은 앵커
  • :visited: 이미 방문한 하이퍼링크

CSS >
a:link{color:blue;}
a:visited{color:gray;}

사용자 동작과 관련된 가상 클래스

"a" 태그에 주로 많이 쓰이며, 조건에 맞는 상황이 되는 요소는 모두 사용 가능

  • :focus: 현재 입력 초점을 가진 요소에 적용
  • :hover: 마우스 포인터가 있는 요소에 적용
  • :active: 사용자 입력으로 활성화된 요소에 적용

a:focus{background-color:yellow;}

a:hover{font-weight:bold;}

a:active{color:red;}

  • :focus
    • 현재 선택을 받는 것
    • 입력 폼 요소에 텍스트를 입력하려고
    • 마우스 클릭 후 커서를 입력 폼 위에 놓으면
    • 입력 폼 요소가 초점을 받는 상태
  • :hover
    • 마우스 커서가 있는 요소에 적용
    • 마우스를 올렸을 때
  • :active
    • 사용자 입력으로 활성화된 요소
    • "a"를 클릭할 때
    • "button"눌렀을 때
    • 순간적으로 활성화

가상 요소

8: 가상 선택자 2

가상 요소 pseudo element

html 코드에 존재하지 않는 구조 요소에 스타일 부여

가상 클래스처럼 문서 내에서 보이지 않지만, 미리 정의한 위치에 삽입

선언: 콜론 사용
CSS3부터 가상 클래스, 가상 요소 구분을 위해 가상 요소에는 :: 더블 콜론 사용
하위 브라우저에서 :: 문법을 지원하지 않는 문제: 상황에 따라 : 사용

:pseudo-element{
property: vlaue;
}

기본적인 가상 요소

  • :before: 가장 앞에 요소 삽입
  • :after: 가장 뒤에 요소 삽입
  • :first-line: 요소의 첫 번째 줄에 있는 텍스트
  • :first-letter: 블록 레벨 요소의 첫 번째 문자

HTML > p | Lorem ipsum dolor ~~ | /p

CSS > p::before{content: "###"}
        p::after{content:"!!!"}
        p:first-line{ ... }
        p::first-letter{ ... }

before와 after 가상요소는 내용이 없는 상태로 생성되기 때문에 내용을 넣기 위해 "content" prop 이용

On Brouser: first-line
"Lorem ipsum dolor ~~"
- 모니터 가로 해상도 따라 내용 변함 -

dolor ~~.

On Brouser: first-letter

9: 구체성

구체성

h1{color:red;}
body h1{color:green;}

같은 요소 선택하는 서로 다른 규칙: 위의 코드는 color:green 적용

구체성: 선택자에 우선으로 적용되는 규칙에 대한 규칙
선택자를 얼마나 명시적으로 선언했느냐를 수치화
구체성의 값이 클수록 우선 적용

0, 0, 0, 0
4개의 숫자 값으로 이루어져 있음
좌측 값부터 비교하며, 좌측 숫자가 클수록 높은 구체성

  • 0, 1, 0, 0: 선택자에 있는 모든 id 속성 값
  • 0, 0, 1, 0: class, 기타 속성, 가상 클래스
  • 0, 0, 0, 1: 선택자의 모든 요소, 가상 요소
  • 0, 0, 0, 0: 전체 선택자
  • 조합자는 구체성에 영향을 주지 않음: >, +, ...

인라인 스타일

CSS > p#page{color:red};
구체성: 0, 1, 0, 1

HTML > p id="page" sytle="color:blue" | ... | /p
구체성: 1, 0, 0, 0
요소에 직접 인라인 방식

결과적으로 p에 color:blue 적용

인라인 스타일이 규칙들 중 가장 큰 구체성을 가짐

important

important 키워드: 별도의 구체성 값X
모든 구체성을 무시하고 우선권 가짐

속성값 뒤 한 칸 공백 + 느낌표 + important

CSS > p#page{color:red !important;}

HTML > p id="page" sytle="color:blue" | ... | /p

p에 important로 인해 color:red 적용

들어가기 전에

상속: 스타일 규칙이 자손요소까지 적용

10: 상속

상속되는 속성

CSS > h1{color:gray;}
HTML > h1| Hello, |em| CSS |/em|h1

Hello CSS

em: 부모인 h1의 color:gray 상속

모든 속성이 상속되지 않음

  • 박스모델은 상속되지 않음
    • margin
    • padding
    • background
    • border
  • 상속되는 속성: 상식적으로 구분 가능

상속되는 속성의 구체성

CSS > *{color:red;}
        h1#page{color:gray;}

HTML > h1 id="page"| Hello, |em| CSS |/em|/h1

선택자 스타일 구체성
전체 선택자 color:red 0,0,0,0
id 선택자 color:gray 0,1,0,1

"em": color:red
상속된 속성은 아무런 구체성을 가지지 못함

들어가기 전에

cascading: 스타일 규칙들이 요소에 적용되는 규칙

  • 폭포처럼 단계적인 의미
  • 모든 스타일 규칙은 단계적 규칙에 따라 요소에 적용

11: 캐스케이딩

cascading 규칙

  1. 중요도(!important)와 (CSS의)출처
  2. 구체성
  3. 선언 순서

CSS의 출처: 제작자, 사용자, 사용자 에이전트(user agent)

  • 제작자: 개발자가 작성한 CSS(almost)
  • 사용자: 웹페이지를 방문하는 일반사용자가 작성한 CSS
  • 사용자 에이전트: 일반 사용자의 환경인 브라우저에 내장된 CSS
추세: 브라우저에서 사용자 스타일을 지원하지 않음

스타일이 적용되는 방식

  1. 스타일 규칙을 모아 중요도가 명시적으로 선언 되었는가
    • 중요도 명시 규칙 우선
    • 중요도 있는 규칙끼리는 아래의 다른 규칙 적용
  2. 스타일 규칙을 출처에 따라 분류
